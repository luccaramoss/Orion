**Volume:** ORION DNA — Camada 1: Fundação

## 1. A FILOSOFIA CENTRAL

Produto no ORION não é feature. É resultado.

Não perguntamos "o que vamos construir?". Perguntamos "qual resultado queremos que o cliente da empresa tenha?"

A resposta a essa pergunta determina o que construímos — e o que deliberadamente não construímos.

---

## 2. OS PRINCÍPIOS DO PRODUTO

### 2.1 Resultado antes de funcionalidade

Cada feature existe para gerar um resultado específico e mensurável. Antes de qualquer desenvolvimento, definimos:

- Qual comportamento do Cliente queremos mudar?
- Como mediremos essa mudança?
- Em quanto tempo esperamos ver o resultado?

Se não conseguimos responder, a feature não entra no roadmap.

### 2.2 A IA deve ser invisível

Inteligência artificial no ORION não é uma seção do menu. Não tem ícone de robô. Não aparece como "Powered by AI".

Ela simplesmente acontece.

O gestor abre o painel e vê: *"Envie uma campanha hoje para esses 214 clientes. Alta probabilidade de gerar R$ 18.000 em vendas."*

Ele não sabe como chegamos a esse número. Ele não precisa saber. Ele clica em enviar.

Isso é IA bem feita.

### 2.3 Simplicidade é trabalho, não ausência de trabalho

A interface mais simples é a mais difícil de construir.

Toda vez que removemos uma opção da tela, alguém teve que tomar uma decisão difícil por baixo dos panos. Toda vez que o usuário não precisa pensar, é porque o produto pensou por ele.

No ORION, simplicidade de interface exige sofisticação de produto.

O cliente da empresa não pensa em canais. Ele compra. Às vezes online, às vezes na loja, às vezes pelo WhatsApp.

O ORION deve enxergar esse cliente como uma única pessoa em todos esses momentos. Se não conseguimos unificar a identidade do cliente através dos canais, não temos produto — temos mais um silo de dados.

### 2.5 Dados são responsabilidade, não commodity

Cada dado que coletamos carrega uma responsabilidade. Coletamos apenas o que usamos. Usamos apenas para gerar valor. Nunca vendemos, nunca cruzamos entre tenants, nunca expostos sem consentimento.

LGPD não é obstáculo. É o padrão mínimo — e devemos ir além dele.

---

## 3. O QUE NUNCA FAREMOS NO PRODUTO

- Adicionar funcionalidade que não tem métrica de sucesso definida
- Construir dashboard com métricas de vaidade (visualizações, acessos) sem conexão com receita
- Criar complexidade de configuração que exija suporte para operação básica
- Lançar feature sem documentação para o usuário final
- Cruzar dados de diferentes tenants sob qualquer hipótese

---

## 4. COMO PRIORIZAMOS

**Prioridade máxima:** features que aumentam retenção de Clientes dos tenants  
**Alta prioridade:** features que aumentam receita dos tenants atribuível ao ORION  
**Média prioridade:** features que reduzem churn dos tenants  
**Baixa prioridade:** features de UX que não impactam as métricas acima  
**Não entra:** features que são tecnicamente interessantes mas não geram resultado mensurável
