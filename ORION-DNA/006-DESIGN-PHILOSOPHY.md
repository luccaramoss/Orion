# ORION DESIGN PHILOSOPHY
## Como Desenhamos Experiências

**Volume:** ORION DNA — Camada 1: Fundação

---

> *"O design ideal é aquele que o usuário não percebe — porque tudo funciona exatamente como ele esperava."*

---

## 1. A FILOSOFIA CENTRAL

Design no ORION não é estética. É clareza.

Nosso usuário principal é o gestor de uma PME — uma pessoa ocupada, que não tem tempo para aprender software e quer resultados imediatos.

Cada tela deve responder a uma pergunta em menos de 3 segundos: "o que eu preciso fazer agora?"

---

## 2. OS PRINCÍPIOS DE DESIGN

### 2.1 Interface leve, motor sofisticado

O usuário não vê algoritmos. Vê recomendações claras em linguagem humana.

O usuário não vê dados brutos. Vê insights prontos para ação.

A sofisticação fica no backend. A leveza fica na tela.

### 2.2 Ação antes de informação

Dashboards que apenas mostram dados sem sugerir ação são bonitos e inúteis.

Cada tela do ORION tem um "próximo passo" óbvio. O usuário sempre sabe o que fazer.

### 2.3 Consistência visual como confiança

Cor, tipografia, espaçamento e comportamento de componentes são consistentes em toda a plataforma.

Inconsistência visual cria desconfiança. O usuário que não sabe o que esperar de um botão azul começa a duvidar do produto inteiro.

### 2.4 Mobile-first, sem exceção

O gestor vai usar o ORION no celular. Provavelmente às 22h enquanto fecha o caixa.

Toda interface é projetada primeiro para mobile. A versão web é a adaptação, não o contrário.

### 2.5 Feedback imediato

Toda ação do usuário tem resposta visual imediata. Carregamentos têm skeleton. Erros têm mensagem humana. Sucessos têm confirmação clara.

O usuário nunca fica sem saber se algo funcionou.

---

## 3. OS DOIS PRODUTOS DE DESIGN

### App do Cliente (B2C)
Interface que o consumidor final vê. Deve transmitir a identidade da empresa tenant (white-label). Simples, rápido, focado em: ver pontos, ver recompensas, receber notificações, interagir com a marca.

### Painel do Gestor (B2B)
Interface da empresa tenant. Deve transmitir clareza e controle. Focado em: ver inteligência de cliente, aprovar campanhas, acompanhar resultados, configurar regras.

---

## 4. O QUE NUNCA FAZEMOS NO DESIGN

- Tela com mais de uma ação primária simultânea
- Jargão técnico visível para o usuário final
- Modal em cima de modal
- Formulário com mais de 5 campos sem paginação
- Dado apresentado sem contexto ou sem ação sugerida
