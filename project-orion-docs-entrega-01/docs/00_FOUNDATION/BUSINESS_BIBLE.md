## 1. Objetivo

Este documento define a base filosófica, estratégica e operacional do Project ORION.

Ele existe para orientar decisões de produto, decisões técnicas, decisões comerciais, decisões de IA, desenvolvimento com Claude Code, entrada de futuros desenvolvedores e evolução da plataforma como SaaS.

Este documento deve ser tratado como a Bíblia do negócio. Ele pode evoluir, mas não deve ser alterado levianamente.

---

## 2. Definição do produto

Project ORION é uma plataforma SaaS white-label de Customer Intelligence para varejo.

Ela combina fidelização, CRM, análise comportamental, automação de campanhas, app do cliente final, painel administrativo, integrações com e-commerce/ERP/PDV/canais de comunicação e inteligência artificial para recomendação, retenção e recompra.

O produto não é apenas um programa de pontos. O produto é uma infraestrutura de relacionamento inteligente entre empresas e seus clientes.

---

## 3. Cliente Zero

O Santo Axé será o primeiro cliente real da plataforma.

Isso significa que ele será o primeiro ambiente de validação, terá as primeiras regras de negócio aplicadas, fornecerá os primeiros dados reais, validará o MVP e demonstrará o valor comercial da plataforma.

Mas o Santo Axé não deve ser codificado como núcleo do produto. O Santo Axé deve ser tratado como uma empresa/tenant dentro do sistema.

---

## 4. Problema que resolvemos

Empresas de varejo perdem receita porque não conhecem profundamente seus clientes, fazem promoções genéricas, não sabem quando o cliente está prestes a recomprar, não recuperam clientes inativos, dependem de ações manuais, não conectam dados de e-commerce/ERP/PDV e oferecem programas de fidelidade superficiais.

Project ORION resolve esse problema criando uma camada inteligente entre dados de compra e ações de relacionamento.

---

## 5. Tese central

O maior ativo de uma empresa de varejo não é apenas seu estoque. É o relacionamento recorrente com seus clientes.

Se a empresa souber quem compra, quando compra, por que compra, o que pode recomprar, o que pode abandonar, qual incentivo funciona e qual canal converte, ela passa a vender melhor, gastar menos e criar mais fidelidade.

---

## 6. O que nunca seremos

Project ORION não deve se tornar um simples app de cupom, sistema genérico de pontos, CRM manual pesado, ferramenta que exige configuração excessiva, plataforma dependente de uma única API, software preso a um segmento ou produto exclusivo do Santo Axé.

---

## 7. O que queremos ser

Queremos ser o sistema de inteligência comercial que ajuda varejistas a aumentar retenção, recompra, ticket médio, frequência, pertencimento, previsibilidade, lifetime value e eficiência em campanhas.

A plataforma deve transformar dados dispersos em ações úteis.

---

## 8. Princípios inquebráveis

### 8.1 White-label desde o início

Toda funcionalidade deve considerar multiempresa. Mesmo o MVP deve ter fundação compatível com tenants.

### 8.2 Integrações desacopladas

Bling, Nuvemshop, WhatsApp, Firebase, OpenAI e qualquer API externa devem ser conectadas por adapters.

### 8.3 Dados são patrimônio

Histórico de compras, preferências, recorrência, ticket médio e comportamento devem ser estruturados desde o início.

### 8.4 IA deve gerar ação

A IA não existe para parecer moderna. Ela deve sugerir campanhas, prever recompra, identificar abandono, personalizar ofertas e melhorar decisões comerciais.

### 8.5 MVP simples, arquitetura séria

O MVP deve ser enxuto, mas nunca improvisado.

### 8.6 Pertencimento antes de desconto

Fidelidade não deve depender apenas de cupom. A plataforma deve criar níveis, status, recompensas, experiências e comunicação personalizada.

### 8.7 Privacidade como padrão

Coleta e uso de dados devem respeitar LGPD, consentimento e boas práticas de segurança.

### 8.8 Documentação é produto

Toda decisão importante deve estar documentada.

---

## 9. Modelo de valor

A plataforma gera valor para o lojista ao aumentar recompra, reduzir abandono, automatizar relacionamento, melhorar segmentação, aumentar ticket médio, criar previsibilidade e reduzir dependência de promoções em massa.

Gera valor para o cliente final ao oferecer recompensas reais, reconhecer comportamento, personalizar ofertas, facilitar compra e criar sensação de pertencimento.

---

## 10. AI Instructions

Quando uma IA ler este documento, ela deve entender que Project ORION é maior que Santo Axé, nunca criar código preso a uma marca, priorizar arquitetura white-label, preservar dados comportamentais, evitar soluções descartáveis, documentar decisões e lembrar que o objetivo final é um SaaS escalável.
