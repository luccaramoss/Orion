Documento: ORION-AI-003  
Versão: 1.0.0  
Status: Oficial

## Regras operacionais

1. Todo código deve ser versionado.
2. Toda configuração sensível deve ficar fora do código.
3. Toda integração deve usar adapter.
4. Toda regra de negócio crítica deve ter teste.
5. Todo módulo deve ter responsabilidade clara.
6. Toda decisão estrutural deve ser registrada em `docs/20_DECISIONS`.
7. Toda mudança relevante deve atualizar documentação.
8. Todo MVP deve ser simples, mas evolutivo.
9. Toda tabela precisa considerar tenant/company.
10. Toda campanha deve ser auditável.

## Regras para Claude Code

Antes de implementar: leia contexto, proponha plano, liste arquivos, explique riscos, implemente em pequenos passos e ensine a testar.

Depois de implementar: resuma o que mudou, liste comandos executados, informe pendências e sugira próxima sprint.
