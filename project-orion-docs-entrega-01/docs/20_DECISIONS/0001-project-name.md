# DECISION 0001 — Nome interno Project ORION

Data: 2026-06-17  
Status: Aprovada

## Decisão

O projeto será chamado internamente de Project ORION.

## Contexto

A primeira validação será feita com o Santo Axé, mas o software deve ser vendável para outras empresas.

## Motivo

Um codinome neutro impede que o produto nasça preso à primeira marca cliente.

## Impacto

Toda documentação e código devem tratar Santo Axé como tenant/cliente, não como núcleo do produto.
