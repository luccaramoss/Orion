## O que é o Project ORION?

Project ORION é uma plataforma SaaS white-label de Customer Intelligence para varejo.

Não estamos construindo apenas um app de fidelidade. Estamos construindo um sistema operacional de retenção de clientes para empresas de varejo.

## Regra central

Toda decisão deve considerar que o Santo Axé é apenas o Cliente Zero.

O software deve ser preparado desde o início para múltiplas empresas, marcas, segmentos, ERPs, lojas virtuais, canais de comunicação, regras de fidelidade e estratégias de IA.

## Como usar este repositório

Este repositório deve servir para:

1. Documentar a visão do produto.
2. Guiar o Claude Code durante o desenvolvimento.
3. Servir como memória permanente da arquitetura.
4. Registrar decisões técnicas e estratégicas.
5. Permitir que novos desenvolvedores entendam o projeto.
6. Evitar que a IA gere código desalinhado com a visão de longo prazo.

## Ordem de leitura para IA

Sempre que uma IA iniciar trabalho no projeto, deve ler:

## Princípios inquebráveis

1. O Santo Axé é Cliente Zero, não o produto.
2. O produto deve ser white-label desde a fundação.
3. Toda integração externa deve ser desacoplada.
4. Nenhuma decisão deve impedir escala futura.
5. Dados comportamentais são o principal ativo da plataforma.
6. IA deve auxiliar decisões, não criar risco operacional sem validação.
7. Código deve nascer documentado e testável.
8. MVP deve ser simples, mas não mal arquitetado.
